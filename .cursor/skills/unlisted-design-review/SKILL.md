# Unlisted Design Review

## Review Checklist

Before finalizing UI changes, check:

- Does this feel credible beside Domain or REA?
- Is the product clearly buy-only?
- Are buttons and inputs appropriately sized?
- Is typography mostly regular/medium outside hero moments?
- Are colors limited to the Unlisted app palette?
- Are shadows, radii, and gradients restrained?
- Is the page useful before it is decorative?
- Does the layout support search, discovery, and buyer action?
- Is there any generic SaaS section that should be removed?
- Is visual density high enough for a serious property platform?

## Common Issues To Flag

- Oversized buttons or pill CTAs
- Bold text everywhere
- Too much cream/gold/luxury styling
- Excessive empty space
- Vague marketing copy
- Rent/sold/commercial concepts creeping in
- Seller-first language
- Cards that look like templates rather than product UI
- Inconsistent mobile and web palette

## Suggested Feedback Format

Use short, direct findings:

```markdown
Findings:
- [High] The search module is too large for a professional property portal.
- [Medium] CTA weight is too heavy outside the hero.
- [Medium] Copy drifts into seller-first language.

Recommended direction:
Tighten control height, reduce font weights, and bring the palette back to app tokens.
```
