# Unlisted Web UX

## Product Shape

The website is a buy-only property search surface. It should behave like a serious real estate marketplace, not a marketing splash page.

## Layout Principles

- Search is the primary product object.
- Keep above-the-fold hierarchy: header, hero/search, supporting search shortcuts.
- Follow familiar property marketplace patterns without copying Domain or REA trade dress.
- Prefer dense, useful layouts over oversized promo sections.
- Let property imagery and search controls carry the page.

## UI Density

- Buttons should be compact and professional.
- Inputs should be clear but not chunky.
- Cards should use restrained radius, subtle borders, and minimal shadow.
- Avoid giant pills, huge CTAs, excessive whitespace, and theatrical gradients.
- Desktop layouts should feel information-rich; mobile can breathe more.

## Website Navigation

Allowed primary nav concepts:
- Buy
- Suburbs
- Insights
- App
- Support

Avoid nav concepts unless requested:
- Rent
- Sold
- Commercial
- New homes
- Seller-focused flows

## Search UX

Search controls should support:
- Suburb, street, and school-zone search
- Filters
- Map view
- Saved search
- Buyer brief creation

Do not add search tabs for rent/sold/off-market. The whole product is already private buying.

## Visual Standard

The page should feel like it can compete with established property portals:
- Utility first
- Calm palette
- Clear hierarchy
- Professional type scale
- No novelty UI for its own sake
